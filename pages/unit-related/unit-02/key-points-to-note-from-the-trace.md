title: Key Points to Note from the Trace
published: true
---

1.  The program execution starts on the first line inside the _public static void main (String\[ \]_ _args__)._ Anything before that is left out of the trace (those are used by compiler).
1.  Make sure that you include the line numbers in your trace so it’s clear in what order the program is running.
2.  Columns
   
    A) Put 1 variable per column; feel free to adjust the size of the font and/or the column width to make the columns fit
    B) Copy the variable's name from the source code and paste it into the column's first row (sometimes called the 'column header') so that the column's name is exactly identical to the variable's name.

        i. It's fine if the name of the variable spills over onto the next line

4.  In a row before a variable is defined, you MUST put a dash (-) in its column to indicate that the variable isn’t in existence yet.
3.  On a row where a variable(s) changes, you MUST format the value in its column very distinctly to make it easy for your instructor to see what's changed (if anything) on each line.
 
    A) If you're using Microsoft Word, there's a style that you can use for this.  In the 'Main' ribbon, look for "0\_Changed" in the Styles panel and use that.
    B)  If you're using something else (such as Apple Pages) make sure that the value is bold faced, larger than normal, and highlighted in yellow.

5.  Output from the program should be listed in the 'Output' table.  
    Make sure to fill this in as you trace through the program (don't fill it in before the trace– the whole point is to practice tracing what the program is doing without having to run it so don't just copy the output from Repl.it or any other IDE you are using. Don't fill it in after the trace – the whole point is to accurately simulate what the computer will do with your program)

    A)  At the same time you are not expected to predict things like round off errors, nor are you expected to do the long division by hand (feel free to use a calculator).
